# 网文源文件输入合约

> EPUB 生成器（`gen_epub_enhanced.py`）与"源数据准备层"之间的契约。
>
> **约定**：生成器按此合约消费文件，不猜源格式；外部准备层（HTML→MD 转换、TXT 拆分、数据清洗）按此合约产出文件。
>
> **范围**：本合约只定义「排版层需要什么」。语义错误（错别字、段落遗漏、章节顺序错乱）由外部负责，生成器不碰。

---

## 1. 核心约定

**一个 `.md` 文件 = 一章**。文件名决定章节顺序（字母序），文件内容决定章节元数据。

### 1.1 文件名

文件名须保证字母序等于阅读顺序：

```
001-重返人间.md
002-魂归来兮.md
003-神秘的邀请函.md
```

推荐前缀 `NNN-` 方案，不要求连续编号，`001-`、`003-` 同样可接受。

**支持**：`001--xxx.md`、`001_xxx.md`、`001-xxx.md`、`卷一-001-灯塔.md`。
**不支持**：文件名中包含真实书名正文（如 `灯塔.md`、`第1章 灯塔.md`）——不会报错，但不参与卷分组自动检测。

### 1.2 最小文件格式

```markdown
# 第一章 灯塔

正文从这里开始。段落之间用空行分隔。

这是第二段。首行不缩进（排版层自动处理缩进）。

这是第三段。

```

以 `# ` 开头的第一行 = 章名。章名即生成器从 H1 提取的内容。

**如果没有 `# ` 开头**：生成器回退到文件名作为章名（不含前缀）。

### 1.3 可选的 YAML Frontmatter

```markdown
---
volume: 卷一
kind: 楔子       # 可选值: 楔子 | 序章 | 引子 | 番外 | 正文(默认)
volume_title: 初入江湖  # 卷副标题，可选，卷封页显示
---

# 第一章 灯塔
```

**字段说明**：

| 字段 | 必填 | 值域 | 默认 | 行为 |
|---|---|---|---|---|
| `volume` | 否 | `卷一` / `第一卷` / `Vol.1` / `第1部` / `第一部` 等 | 无（不分卷） | 同卷章节归入同一 NCX 父节点 |
| `kind` | 否 | `楔子 \| 序章 \| 引子 \| 番外 \| 正文` | `正文` | 影响 TOC 区域位置 |
| `volume_title` | 否 | 任意字符串 | 无 | 卷封页上显示在卷号下方 |

**不写 frontmatter 的 `.md` 文件同样合法**——生成器会从 `# 第一章 灯塔` 自动解析章号。

## 2. 卷分组：三种约定（按优先级）

### 方案 A：frontmatter（推荐，最显式）

```markdown
---
volume: 卷一
---

# 第一章 灯塔
```

所有 frontmatter 中 `volume` 值相同的文件归入同一卷。

### 方案 B：文件夹分层

```
novel/
├── 卷一/
│   ├── 001-灯塔.md
│   └── 002-守夜人.md
└── 卷二/
    ├── 001-远行.md
    └── 002-风暴.md
```

文件夹名即卷名。生成器按文件夹分组，文件夹内按文件名排序。

### 方案 C：文件名前缀

```
卷一-001-灯塔.md
卷一-002-守夜人.md
卷二-001-远行.md
```

`^(.+?)[-_\s]+\d+[-_].*\.md$` 提取前缀作为卷名。

### 无卷（默认）

```
001-灯塔.md
002-守夜人.md
```

扁平 TOC，无卷封页。**建议大多数网文使用**——500+ 章一卷全平铺或按网站已有分卷走方案 A。

## 3. 特殊元素契约

### 3.1 楔子 / 序章 / 引子

TOC 位置：第一卷之前的「前置区」，保留独立区域。章首页无章号（不显示"第一章"）。

文件示例：

```markdown
---
kind: 楔子
---

# 楔子

正文…
```

约定：楔子 / 序章 / 引子不得重复（单本书各最多一个，生成器不强制校验）。

### 3.2 番外

TOC 位置：最后一卷之后的「附录区」。章首页同样无章号。

```markdown
---
kind: 番外
---

# 番外 · 中秋夜
```

番外可多个，文件名顺序即阅读顺序。

### 3.3 嵌入元数据（章末）

生成器不额外处理。如需在章末显示字数、创作日期等，建议写在正文末尾的 `>` 引用块中：

```markdown
正文结束。

> 本章完 ·
```

排版层将该引用块渲染为 olive 色小字（stone 0.85em），与正文形成视觉分离。

## 4. 生成器负责的归一化

以下排版的归一化由生成器做，**外部不碰**。越界说明：

| 任务 | 负责方 | 理由 |
|---|---|---|
| 章号解析（从"第一章""第1章""Chapter 1"提文字） | 生成器 | 排版层 4 档字号自适应需要 |
| 段首缩进（首段缩进 0，后续缩进 2em） | 生成器（CSS） | 排版层 |
| 悬挂标点 + 避头尾 | 生成器（CSS） | 排版层 |
| 场景切换符 → ❦ | 生成器 | `---` / `***` / `--------` / 连续空行都归一 |
| 中文引号排他性（行首禁 `」』。，；！？`） | 生成器（CSS） | 排版层 |
| 中英之间插空格 | 生成器 | 排版层归一 |
| `...` → `……`、`--` / `—` → `——` | 生成器（`--no-punct-fix` 可关） | 排版层，每个站点重写不可复用 |

## 5. 源材料处理指南（外部准备层参考）

生成器不参与以下步骤，但本合约约定"标准做法"：

### 5.1 HTML → MD（爬虫来源）

1. 清洗 HTML：去掉无关导航、广告、评论、脚本、CSS
2. 提取正文文本区域（`<article>`、`<div class="content">` 等）
3. 合并分页（如果一篇文章分多页）
4. 输出为纯文本或极简 Markdown

### 5.2 TXT → MD（整本小说文本）

推荐从以下模式拆章，按优先级：

```python
# 最常见——按"第X章""第X节""Chapter X"
^第[一二三四五六七八九十百千万零0-9]+[章节篇集部卷回]\s*
^第\d+[章节篇集部卷回]\s*
^Chapter\s+\d+
^CHAPTER\s+\d+

# 数字编号
^(\d{1,3})[、\.\s]+

# 卷内章节
^[卷部集][一二三四五六七八九十]+[ 　]*第[一二三四五六七八九十零0-9]+[章节回]
```

拆分后的每章保存为独立 `.md` 文件，文件名前缀 `001-` 起。

### 5.3 章节确认

最终产出应满足：

- 每个 `.md` 文件以 `# 章名` 开头
- 文件数 = 章节数（含楔子 / 番外）
- 文件名字母序 = 阅读顺序
- 无空文件、无重复内容
- frontmatter 中的 `volume` / `kind` 与内容一致

## 7. 错误防范

| 场景 | 生成器行为 | 建议外部准备 |
|---|---|---|
| 空文件 | 跳过，警告 | 检查清除 |
| 章名重复 | 不校验 | 外部排重 |
| volume 拼写不一致（"卷一"vs"第一卷"） | 视为不同卷 | 统一写法 |

## 8. 验收标准（Verification Checklist）

以下标准可用 Python 脚本自动化检验。每条标准标注（强制 / 警告），强制项不通过需修，警告项提示但可放行。

| # | 检查项 | 等级 | Python 检验方式 |
|---|---|---|---|
| 1 | 每个 `.md` 文件可读（UTF-8, 非空） | 强制 | `os.listdir()` → 逐一 `open().read()` |
| 2 | 文件名格式 `NNN-*.md` 或 `NNN_*.md` | 强制 | `re.match(r'\d+[-_].+\.md', fname)` |
| 3 | 第一行以 `# ` 开头（H1 章名） | 强制 | `content.split('\n')[0].startswith('# ')` |
| 4 | 文件名字母序 = 预期阅读顺序（无随机漂移） | 强制 | `sorted(fnames) == fnames` |
| 5 | H1 值 = 文件名中`-`后的部分（近似校验，可容忍部分差异） | 警告 | 检查 H1 是否包含文件名关键词 |
| 6 | 所有文件的 `volume` 值拼写一致（无"卷一"vs"第一卷"混用） | 强制 | 读取 frontmatter 中的 `volume`，检查唯一值集合 |
| 7 | `kind` 值来自合法集合：`正文\|楔子\|序章\|引子\|番外` | 强制 | 同上，`kind in {allowed}` |
| 8 | 楔子 / 序章 / 引子 每种最多出现一次 | 警告 | 计数 |
| 9 | 无重复 H1 章名（不同文件相同章名） | 警告 | 收集所有 H1 检查是否有重复 |
| 10 | 内容中不包含 `<div`、`<a ` 等常见残留 HTML 标签（从 HTML 来源转换时） | 警告 | `re.search(r'<(div\|a\s\|script\|style\|nav)', content)` |
| 11 | 文件总数 ≥ 1 | 强制 | `len(files) >= 1` |
| 12 | frontmatter YAML 格式正确（若有） | 强制 | 尝试 `yaml.safe_load()`，捕获异常 |

### 验收脚本模板

```python
#!/usr/bin/env python3
"""验收脚本：检查小说源文件是否符合输入合约。"""
import os, sys, re, yaml

CHECKS = []

def check(level, name, condition, detail=""):
    """记录检查结果。level: "强制" 或 "警告" """
    CHECKS.append({"level": level, "name": name, "pass": bool(condition), "detail": detail})

def main():
    import glob
    md_dir = sys.argv[1]
    files = sorted(glob.glob(os.path.join(md_dir, "*.md")))
    
    check("强制", "至少1个文件", len(files) >= 1, f"找到 {len(files)} 个文件")
    
    # ... 逐条检查 ...
    
    # 输出结果
    failed = [c for c in CHECKS if c["level"] == "强制" and not c["pass"]]
    warned = [c for c in CHECKS if c["level"] == "警告" and not c["pass"]]
    print(f"通过: {len(CHECKS) - len(failed) - len(warned)}/{len(CHECKS)}")
    print(f"强制失败: {len(failed)}")
    print(f"警告: {len(warned)}")
    if failed:
        print("\n必须修复:")
        for c in failed:
            print(f"  ✗ {c['name']}: {c.get('detail', '')}")
        sys.exit(1)

if __name__ == "__main__":
    main()
```

## 附录 A：使用方式（给 Claude 的提示词）

> 以下描述 Claude Code 应如何根据本合约从原始材料生成符合规范的源文件。

**核心原则：不要直接用 Read/Write 工具逐文件处理。先写脚本，再跑验收，迭代到全绿。**

### 工作流

```
① 分析原始材料结构
    ↓
② 编写一次性 Python 转换脚本
    ↓
③ 编写一次性 Python 验收脚本（基于 §8 Checklist）
    ↓
④ 跑转换脚本 → 跑验收脚本
    ↓  (失败)
⑤ 分析失败原因 → 改转换脚本 → 回到 ④
    ↓  (全部强制项通过）
⑥ 完成
```

### 第一步：分析

- 如果是 HTML：打开 1-3 个样本文件，识别正文容器（`<div class="content">` 等）、需要去掉的噪声（广告、导航、评论）、分页关系
- 如果是 TXT：打开头部几十行，确认章节分隔符的正则模式
- 如果是已有 MD：检查文件命名是否规范、H1 是否完整

### 第二步：写转换脚本

编写的脚本应当：

- **针对这一次的源材料**（不是通用工具）
- 可重复运行（幂等——每次覆盖输出目录）
- 产出直接符合 §1-§3 的文件格式

HTML 来源的脚本核心逻辑示例：

```python
# 一次性转换脚本，针对某个小说站
import os, re
from bs4 import BeautifulSoup

OUTPUT = "./output-md/"
os.makedirs(OUTPUT, exist_ok=True)

html_files = sorted(os.listdir("./source-html/"))

for i, fname in enumerate(html_files):
    with open(f"./source-html/{fname}") as f:
        soup = BeautifulSoup(f.read(), "html.parser")
    
    # 提取正文 —— 分析阶段确定的容器选择器
    article = soup.select_one("div.article-content")
    title = soup.select_one("h1.chapter-title").text.strip()
    
    # 清洗
    for tag in article(["script", "style", "nav", ".ad"]):
        tag.decompose()
    
    text = article.get_text("\n\n").strip()
    
    # 输出
    with open(f"{OUTPUT}/{i+1:03d}-{title}.md", "w") as out:
        out.write(f"# {title}\n\n{text}\n")
```

### 第三步：写验收脚本

验收脚本基于 §8 验收标准的模板，补充本批材料的特殊检查（如"HTML 来源没有残留标签"）。

### 第四步：迭代

- **跑转换脚本 → 验收脚本 → 看哪项没通过 → 改转换脚本 → 重新跑**
- 验收脚本的失败项直接告诉你哪里需要修
- 不要在失败项上蒙混过关——如果一段逻辑写不对，改正则而不是删数据
- 验收全部强制项通过后才算完成

### 第五步：收尾

- 通知用户输出目录和统计（文件总数、总字数、有无警告）
- 转换脚本和验收脚本可以扔掉，也可以保留在 `scripts/prepare/` 目录下供复现
